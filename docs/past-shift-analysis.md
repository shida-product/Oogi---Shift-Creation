# 過去シフト（2〜4月）の実態調査レポートと重要ロジック

2026年3月のシステム改修にあたり、過去に手動で作成された2月〜4月の確定シフト（CSV）と、同月に入力されていた希望休データ（DB）を突合して分析した結果の備忘録です。AIの自動生成ロジックの改善（S2・S3ルールの調整など）の根拠となっています。

## 1. 希望休は「100%」最優先で反映
- 過去3ヶ月間で提出された全スタッフの希望休（計213件）は、実績シフト上で「100%」完全に休みとして反映されていました。
- **重要ロジック:** シフト作成における絶対的プライオリティの最上位は「希望休の完全反映」であり、AIアルゴリズムにおいても、後述する連休ルール（S3など）よりも優先されるべき絶対条件（ハード制約）です。

## 2. 信太さんの「絶対2連休にする（S3）」ルールの実態（緩和の根拠）
- 初期要件では「信太さんの公休は2連休ペアで配置する」となっていましたが、実態としては**3ヶ月で単発休みが9回**発生していました。
- **重要ロジック:** 希望休を100%叶えつつ店舗の人数要件（H1）を満たすためには、信太さんを全て2連休で組むことは数学的・実務的に不可能であり、人間側も「単発休み」を柔軟に使ってパズルを解いていました。
- **対応:** AIのロジックでも、S3の縛りを緩和し「ペア連休は月2回程度・残りは単発休みを許容する」動作に変更しました。

## 3. 小野さんの「単発休みと日曜絡みの2連休（S2）」ルールの実態
- 小野さんの休みは「2連休にする場合は日曜に隣接させること（日曜＋月曜など）。それ以外の平日2連休は不可とし、残りは孤立させる（単発休み）」というS2ルールがありました。
- **重要ロジック:** 実績データでは、まさにこの通り「日曜絡みの2連休」と「単発休み」のみで構成されており、平日での2連休はゼロ（ルール違反0件）でした。
- 小野さんの休みが自然と「単発」に分散するため、穴埋めに入る徳永さんの勤務ペース（連勤防止）も自然と保たれる設計になっており、極めて完成度の高いルールとして機能していました。

## 4. 「法定休日」の扱い
- 勤怠区分で「法定休日」がマークされていたのは、過去3ヶ月で小野さんの「日曜日」に数回のみでした。
- **重要ロジック:** 労働基準法対応（週1日の法定休日確保）のために事務上のマーキングとして使われているのみであり、シフト生成のパズルアルゴリズム上は、単純な「所定休日（通常の休み）」と全く同じ「休み（出勤不可日）」として扱って構わないことが確定しました。追加の生成ロジックは不要です。

## 5. 連勤日数の上限（H3）
- 要件では「最大5連勤まで」としていますが、実態データではどうしても人が足りないタイミング等で小野さんや信太さんに「6連勤」が発生していました。
- **対応:** 基本のターゲット上限は「5連勤以下」のままでAIに組ませますが、将来的にエラーでシフトが全く生成できなくなるような場合は、連勤MAXを「6」に一段階引き上げて試行する余地があることがわかりました。

## 事務パートの連勤・連休に関する分析（2月〜4月）

### 1. 連勤の実態（現行の「最大5連勤ルール」の妥当性）
*   **木庭**: 最大5連勤（1〜3連勤が主力、4〜5連勤も月2回程度生じる）
*   **諫早**: 最大4連勤（1連勤が主力、次いで2〜3連勤）
*   **中村**: 最大4連勤
*   **本庄**: 最大3連勤
*   **結論**: 過去3ヶ月間で全事務パートにおいて「6連勤以上」したケースは**ゼロ**でした。したがって、現在AIに設定している**「最大5連勤でブロックする」というハード制約は、過去の実績と完全に一致**しており極めて妥当です。

### 2. 連休の実態（シフトの間隔）
*   **レギュラー（木庭・諫早）**: 「単発休み」が圧倒的に多く（月15〜17回）、次いで「2連休」（月6〜9回）という分布になっています。
*   **少日数・穴埋め（中村・本庄）**: 勤務日数が少ないため、数日おきの勤務となり「5連休以上」のブランクも頻繁に発生しています。
*   **結論**: 当初案にあったような「事務パートは必ず○連休にする」「シフト間隔を空けすぎない」といった特別な連休ルールは実態には存在しませんでした。先ほど実装した「目標日数に対してペース配分（進捗率）を計算し、ペースが早過ぎたら穴埋めスタッフに譲る」というアルゴリズムにより、この「レギュラーは数日おきに働き、穴埋めスタッフは間隔が空く」という実態を自然にシミュレーション可能です。追加の連休制約を設ける必要はありません。
